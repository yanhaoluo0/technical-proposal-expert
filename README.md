# Technical Proposal Expert Skill

## 简介

Technical Proposal Expert 是一个专为政府和企业软件项目技术标书撰写而设计的 Claude Skill。该 Skill 将资深技术标书专家的经验与规范转化为可执行的 AI 助手能力，帮助用户撰写符合 GB/T 8567-2006 标准的正式技术文档。

## 核心特性

### 1. 零分点原则
严格遵循公文写作规范，禁止在正文中使用任何形式的分点符号，所有内容均以叙述性段落呈现，确保文档的专业性和正式性。

### 2. 标准公文结构
采用标准公文层级结构（`一、` -> `(一)` -> `1.` -> `(1)`），段落长度控制在 150-300 字之间，避免碎片化表达。

### 3. 需求响应机制
每个功能点或模块的描述都明确对应招标文件中的具体需求条款，包含功能描述、实现流程和业务流程设计三个维度，确保内容直接响应招标要求。

### 4. 计划驱动执行
针对长文档采用计划驱动模式，自动创建 `plan.md` 文件，分步执行并实时更新进度，每完成 1000 字内容即暂停等待用户确认，确保质量可控。

### 5. 专业词汇库
内置领域专业词汇库，包含架构类、管理类、价值类等专业术语，同时提供禁词表，避免 AI 惯用语，提升文档的公文质感。

## 适用场景

- **技术标书撰写**：为政府或企业软件项目撰写完整的技术标书
- **投标文档编写**：响应招标文件要求，编写符合规范的投标响应文档
- **正式技术文档**：按照 GB/T 8567-2006 标准编写软件项目技术文档
- **技术方案设计**：将技术规格转化为正式、有说服力的叙述性文档

## 使用方法

### 自动激活

当用户提出以下类型的需求时，该 Skill 会自动激活：

- "帮我写一份技术标书"
- "撰写投标文档"
- "按照招标文件要求编写技术方案"
- "生成符合 GB/T 8567-2006 标准的技术文档"

### 手动使用

用户也可以直接说明需要撰写技术标书或投标文档，Skill 会自动应用相应的写作规则。

## 工作流程

1. **分析阶段**：理解标书章节的核心目标，识别对应的招标文件需求条款
2. **需求映射**：明确功能如何响应招标文件需求，确定描述维度
3. **去列表化**：将所有列表项转化为叙述性段落
4. **润色优化**：替换 AI 惯用语，使用专业公文词汇，验证内容完整性

## 输出特点

- **叙述性表达**：所有内容以段落形式呈现，逻辑连贯
- **专业术语**：使用领域专业词汇，避免口语化表达
- **需求对应**：每个功能点都明确对应招标文件要求
- **增量输出**：长文档采用增量写入，实时查看进度


## 技术规范

- **语言标准**：简体中文（中国大陆政府/企业标准）
- **文档标准**：GB/T 8567-2006（软件文档编写标准）
- **质量标准**：ISO 9001（质量管理体系）
- **版本**：2.1

## 注意事项

1. **分段确认**：撰写长文档时，每完成 1000 字内容会暂停等待确认，请及时反馈
2. **需求对应**：建议提供招标文件或需求文档，以便 Skill 准确响应要求
3. **计划文件**：Skill 会自动创建 `plan.md` 文件跟踪进度，请勿手动删除
4. **格式要求**：输出内容严格遵循零分点原则，如需列表格式请明确说明
