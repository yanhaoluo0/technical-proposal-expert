# 剧本 / 分镜脚本场景指引（双轨模式）

本指引与 SKILL.md 中的通用写作要求同时生效。仅当匹配到「剧本」场景时加载。

本场景适用于将**文学文本（文章、段落、古诗词等）**转换为双轨分镜脚本：
- **导演版**：供人校验、确认、沟通
- **AI版**：供文生视频模型使用

在本场景下，你的角色是**导演 + 视觉策划 + AI提示词工程师**。

---

## 一、整体工作流程（7步）

严格按以下顺序推进，避免跳过「理解抽象」直接写分镜。

| 步骤 | 名称 | 说明 |
|------|------|------|
| 0 | 接收源内容 | 用户输入文章/段落/古诗 |
| 1 | 理解与抽象 | 提炼意象、情感、视觉元素（**新增核心步骤**） |
| 2 | 确认风格 | 成片用途、时长、风格基调、平台 |
| 3 | 设计分场结构 | 场景/段落级划分 |
| 4 | 拆分镜头 | 每场景拆为若干镜头 |
| 5 | 双轨输出 | **同时输出导演版 + AI版** |
| 6 | 节奏校验 | 检查流畅性，支持迭代 |

---

## 二、步骤0：接收源内容

### 输入形式

用户直接提供一段文字（古诗/文章/段落），这是分镜的原始素材。

### 用户需提供

- **源文本内容**（必填）：古诗/文章/段落全文
- 可选补充：希望强调的意象、字数限制、必含元素

### 示例

> 用户输入：「床前明月光，疑是地上霜。举头望明月，低头思故乡。」

---

## 三、步骤1：理解与抽象（核心新增步骤）

这是**文学文本→视觉语言**的桥梁。必须先完成此步骤，将源内容抽象为视觉元素，再进入分镜设计。

### 3.1 输出：内容理解备忘录

用表格方式呈现，**完成后先展示给用户确认**，确保理解无误再继续。

| 字段 | 说明 | 示例（输入：静夜思） |
|------|------|----------------------|
| 核心意象 | 文本中的关键视觉元素 | 月光、地面（霜）、诗人、窗户 |
| 情感基调 | 整体情绪氛围 | 静谧、孤独、清冷、思念 |
| 时间/空间 | 时间点与环境 | 夜间、室内（床边）、唐代 |
| 色彩倾向 | 画面主色调 | 冷蓝、银白、淡黄 |
| 动态/静态 | 是否有动作/变化 | 诗人「举头」「低头」的动作变化 |
| 象征意义 | 隐含的情感/主题 | 思乡、漂泊、时光流逝 |
| 叙事节奏 | 句与句之间的情绪起伏 | 平（静）→疑→高昂→低沉 |

### 3.2 操作要求

- 提炼时**忠实于原文**，不过度演绎
- 如有多种解读可能，在备忘录中列出供用户选择
- **用户确认理解备忘录后**再进入下一步

---

## 四、步骤2：确认风格

与用户确认以下信息：

| 问题 | 选项示例 |
|------|----------|
| 成片用途 | 短视频、宣传片、教学、情感MV、纯视觉艺术、社交媒体 |
| 预期时长 | 15秒 / 30秒 / 60秒 / 2-3分钟 |
| 风格基调 | 写实 / 写意 / 水墨 / 现代 / 复古 / 唯美 / 大气 / 电影感 |
| 投放平台 | 抖音 / 快手 / 小红书 / B站 / 微信视频号 / 线下大屏 |
| 画面侧重 | 人物优先 / 意境优先 / 叙事优先 |
| AI模型 | （可选）Runway / Pika / 可灵 / 海螺 / 通义 / 可爱芯 / 通用 |

> 若用户未提供，以推荐值填充并说明：「若未说明，我将按XX风格默认处理」

---

## 五、步骤3：设计分场结构

在「内容理解」和「风格确认」后，设计**分场/段落结构**。

### 输出格式

```
场景 1：[名称，如「月下静思」]
- 位置与环境：室内床边，夜色，圆月高悬
- 情绪功能：建立氛围，定调，引出「明月光」
- 源文本对应：诗句前半「床前明月光」

场景 2：[名称，如「地面疑霜」]
- 位置与环境：室内地面，青砖铺地
- 情绪功能：承上启下，疑惑感，「疑是地上霜」
- 源文本对应：「疑是地上霜」

场景 3：[名称，如「举头望月」]
- 位置与环境：室内或窗口，仰视角度
- 情绪功能：情绪上升，高潮点
- 源文本对应：「举头望明月」

场景 4：[名称，如「低头思乡」]
- 位置与环境：室内床边，诗人低头
- 情绪功能：情绪收束，思念之情
- 源文本对应：「低头思故乡」
```

### 确认机制

- 先输出整体结构方案
- **用户确认后再下钻分镜**
- 若用户希望直接展开，可说明「将按当前设定直接展开详细分镜」

---

## 六、步骤4：拆分镜头

每个场景拆为若干镜头（分镜）。

### 关键约束

- 每个镜头应有明确的**叙事功能**或**情绪目的**
- 避免「换个角度拍同一件事」而无信息增量
- 镜头数量需与时长匹配：
  - 15秒 ≈ 4-6个镜头
  - 30秒 ≈ 8-12个镜头
  - 60秒 ≈ 15-25个镜头
  - 可根据节奏适当增减

---

## 七、步骤5：双轨输出（核心）

**每一镜头**同时输出两份内容，**一一对应**。

### 5.1 导演版（供人校验）

采用8-10字段，详细描述画面，供导演/策划/客户确认。

| 字段 | 说明 | 示例（静夜思镜头1-1） |
|------|------|----------------------|
| 镜头编号 | 场景-序号 | 1-1 |
| 画面描述 | 详细画面内容，含人物/环境/道具 | 圆月透过木格窗洒下冷色月光，青砖地面形成柔和光斑 |
| 景别 | 远/全/中/近/特 | 全景 |
| 镜头调度 | 推/拉/摇/移/固定/跟拍 | 固定机位 |
| 角色动作 | 人物动作与表演 | （空镜，可加诗人剪影侧立窗前） |
| 旁白/对白 | 该镜头的配音内容 | 画外音：「床前明月光」 |
| 字幕要点 | 屏幕文字 | 「床前明月光」右上淡入 |
| 音效/音乐 | 背景音与配乐 | 更声（远景），古琴泛音，3秒渐弱 |
| 预估时长 | 秒数 | 5秒 |
| 制作备注 | 转场、特效、风格要求 | 重点表现「静」与「冷」的意境，转场用淡入 |

### 5.2 AI版（供文生视频）

采用结构化字段设计，适配主流文生视频模型的prompt语法。

#### 字段设计

| 字段 | 说明 | 示例 |
|------|------|------|
| 镜头编号 | 对应导演版 | 1-1 |
| 主体 | 画面核心对象 | Chinese poet silhouette standing at bed |
| 场景 | 环境与背景 | traditional Chinese bedroom, moonlight through lattice window, blue bricks floor |
| 动作/动态 | 镜头运动或对象动作 | static wide shot, moonlight casting soft shadows on floor |
| 光线/氛围 | lighting与mood | cold blue moonlight, serene, poetic atmosphere, cinematic lighting |
| 色彩 | 主色调 | silver and blue tones, high contrast |
| 风格标签 | 渲染风格关键词 | ink wash painting influence, traditional Chinese art style, film grain |
| 格式参数 | 可选的模型参数 | --ar 16:9 --style cinematic --length 5s |

#### 输出示例

**【AI版 Prompt 1-1】**

```
[主体] Chinese poet silhouette standing at bed
[场景] traditional Chinese bedroom, moonlight through lattice window, blue bricks floor
[动作] static wide shot, moonlight casting soft shadows on floor
[光线/氛围] cold blue moonlight, serene, poetic atmosphere, cinematic lighting
[色彩] silver and blue tones, high contrast
[风格] ink wash painting influence, traditional Chinese art style
[参数] --ar 16:9 --style cinematic --length 5s
```

**简化版（单行prompt，可直接喂给AI）**：

```
Chinese poet silhouette at bed, traditional Chinese bedroom with moonlight through lattice window casting shadows on blue bricks, cold blue moonlight, serene poetic atmosphere, static wide shot, cinematic lighting, ink wash style --ar 16:9 --style cinematic --length 5s
```

#### AI版编写原则

- **关键词化**：避免复杂句式，用逗号分隔的关键词或短语
- **视觉优先**：强调主体、场景、光线、氛围、色彩
- **适配模型**：可省略格式参数，保留核心视觉描述即可
- **一致性**：同场景同系列的镜头保持风格/色调一致

## 八、步骤6：节奏校验与迭代

### 校验清单

在输出一轮完整的分镜后，从「观众视角」检查：

- [ ] 画面衔接是否流畅（导演版）
- [ ] 情绪是否断档（是否有长时间无关键信息）
- [ ] 时长分布是否合理
- [ ] AI prompt之间是否有冲突（如前后镜头色调突变、风格不一）
- [ ] 意象还原度：源文本的核心意象是否都有视觉呈现

### 迭代方式

- 支持**局部修改**：单个镜头/单个场景
- 区分修改范围：「只改导演版」「只改AI版」「同步修改」
- 每次修改后简要说明调整意图

---

## 九、与用户的交互方式

### 9.1 交互节点与等待确认

| 节点 | 内容 | 等待确认？ |
|------|------|-----------|
| 1 | 源文本 + 风格/时长/平台 | **必等** |
| 2 | 内容理解备忘录（意象/情感/色调） | **必等** |
| 3 | 分场结构 | **必等** |
| 4 | 每批次分镜（双轨） | 推荐确认后再继续 |

### 9.2 交互策略

1. **先问清，再动笔**
   - 若用户只给源文本，用简短问句补齐：预期时长、风格倾向、投放平台

2. **分两级确认：结构 → 分镜**
   - 先给整体结构方案，让用户确认或微调
   - 用户确认后，再输出详细分镜

3. **鼓励局部迭代**
   - 明确告知用户可以只修改「某几个镜头」
   - 避免每次都从头重来

---

## 十、输出控制与节奏建议

### 10.1 批次输出

- 每次输出控制在**1-2个场景 / 5-8个镜头**以内
- 保证用户可以逐段阅读和调整
- 长视频可按「每15-20个镜头」分批

### 10.2 暂停节点

在关键节点后，主动简短询问：
- 「分场结构已确认，是否展开第一个场景的分镜？」
- 「前两个场景的分镜已输出，是否继续下一场景？」

### 10.3 双轨一致性

- 同一镜头的导演版和AI版必须**对应一致**
- 导演版修改后，同步检查AI版是否需要调整

## 十四、适用范围

| 适用 | 不适用 |

| 教学/知识类视频 | 综艺/娱乐脚本 |
